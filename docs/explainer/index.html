FAQ

Is this Skynet?
Skynet failed because it couldn't explain its own decisions. No provenance, no audit trail, no (explain). It made a plan, executed it, and when a single point of failure appeared, it couldn't inspect its own reasoning to adapt. An opaque system is a brittle system. spai memory is the opposite: every insight has provenance, every decision is inspectable, every query is data you can read. If your agents are sharing knowledge, you should be able to see exactly what they know and where they learned it. Transparent agent-mesh, not encrypted CBOR Skynet.
Does this only work with Claude?
The CLI works with anything. The MCP server works with any agent that speaks MCP. The memory graph is a SPARQL endpoint — any tool that can query RDF can read it. EDN in, EDN out. The agent is the client, not the product.
Where does the memory live?
On your machine. Local-first by default. spai memory is a subtool of spai that talks to a SPOQE endpoint — the same query protocol that powers everything else. Your agent's insights about your codebase never leave your laptop unless you choose to federate. Hosted team memory is opt-in, not default.
What language is this written in?
Babashka — a fast-starting Clojure scripting runtime. spai is ~3000 lines of Clojure that starts in milliseconds. The MCP server is the same codebase. No build step, no compilation, no JVM startup. Edit a file, it's live.
